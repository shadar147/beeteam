# BeeTeam Foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up the full BeeTeam monorepo skeleton (Docker infra, Rust/axum API, Postgres schema + seed, Next.js + design tokens, OpenAPI→TS codegen) and prove the stack end-to-end: the Next.js app reads `GET /v1/health` through a generated typed client.

**Architecture:** Monorepo with `/api` (Rust workspace: `bt-domain` types, `bt-db` sqlx + migrations, `bt-api` axum bin) and `/web` (Next.js 14 App Router). Postgres + MinIO run via docker-compose. axum exposes an OpenAPI document via `utoipa`; the web build generates TS types from it with `openapi-typescript` and calls the API through `openapi-fetch`. Rust is the single source of truth for the API contract.

**Tech Stack:** Rust (axum 0.7, sqlx 0.8/Postgres, utoipa 5, tower-http 0.6, tokio, thiserror, uuid, chrono, dotenvy), Next.js 14 + TypeScript + Tailwind CSS 3 + shadcn/ui + TanStack Query v5 + openapi-fetch, Vitest + Playwright, PostgreSQL 16 + MinIO via Docker.

This plan is the first of several per-slice plans for the Core 1-2-1 spec (`docs/superpowers/specs/2026-05-29-beeteam-core-design.md`). It implements **build-order slice 1 (Foundation)**. Auth, TeamList, Profile/History, MeetingDrawer, Goals, Files, and Calendar each get their own plan.

> **Note on sqlx compile-time checks:** Foundation uses the runtime query API (`sqlx::query(...)`) for the seed and a DB-less health handler, so no live DB is needed at build time. Later feature plans adopt `query_as!` with an offline cache (`cargo sqlx prepare`, committed `.sqlx/`). This is intentional to keep the foundation build unblocked.

- [ ] **Step 2: Write `AppError` (`api/crates/bt-api/src/error.rs`)**

```rust
use axum::http::StatusCode;
use axum::response::{IntoResponse, Response};
use axum::Json;
use serde_json::json;

#[derive(Debug, thiserror::Error)]
pub enum AppError {
    #[error("not found")]
    NotFound,
    #[error("bad request: {0}")]
    BadRequest(String),
    #[error("unauthorized")]
    Unauthorized,
    #[error(transparent)]
    Db(#[from] sqlx::Error),
}

impl IntoResponse for AppError {
    fn into_response(self) -> Response {
        let (status, detail) = match &self {
            AppError::NotFound => (StatusCode::NOT_FOUND, self.to_string()),
            AppError::BadRequest(_) => (StatusCode::BAD_REQUEST, self.to_string()),
            AppError::Unauthorized => (StatusCode::UNAUTHORIZED, self.to_string()),
            AppError::Db(e) => {
                tracing::error!("db error: {e:?}");
                (StatusCode::INTERNAL_SERVER_ERROR, "internal error".to_string())
            }
        };
        (status, Json(json!({ "error": detail }))).into_response()
    }
}

pub type AppResult<T> = Result<T, AppError>;
```

```rust
use axum::routing::get;
use axum::Router;
use sqlx::PgPool;
use tower_http::cors::{Any, CorsLayer};
use tower_http::trace::TraceLayer;

use crate::routes;

#[derive(Clone)]
pub struct AppState {
    pub pool: PgPool,
}

/// Build the application router. Pure function of state — used by tests too.
pub fn build_router(state: AppState) -> Router {
    let cors = CorsLayer::new()
        .allow_origin(Any)
        .allow_methods(Any)
        .allow_headers(Any);

    Router::new()
        .route("/v1/health", get(routes::health::health))
        .layer(TraceLayer::new_for_http())
        .layer(cors)
        .with_state(state)
}
```
